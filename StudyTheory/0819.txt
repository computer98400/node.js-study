//npm 시작.
node package manager

npm은 노드의 전체적인 관리를 해주는 매니저라 생각하면된다.(CD게임에서 mode 매니저랑 비슷한 느낌이다.)

package.json의 경우는 환경설정이라 보면 되겟다. 환경설정인 만큼 가장 중요한 내용을 담고 있다고 봐도된다.

npm은 타사의 패키지를 다운받을 수 있게 해두었는데
이때 사용 되는 명령어는 npm install이다.

추가로 원하는 패키지가 있을 시엔 npmjs.com을 들어가서 확인하면 되겠다.

패키지를 다운하기에 앞서 종속성에 대해서 이해를 해야한다.

여러 패키지를 다운받게 될시에 해당 패키지가 어디에 종속되는가 == 어디에 사용할 패키지인가
를 정확히 해야한다.

예시로 나온 nodemon의 경우 개발분야에 유용하게 사용하기 위한 패키지로써 실제 서버에선 사용되지 않는다.
이때 사용되는 명령어는 npm install nodemon --save-dev이다.

패키지를 설치할시 package.json에 해당 패키지가 추가된다( 버전과 함께)


기본적으로 npm install을 입력할 경우 업데이트를 한다고 생각하면된다.
package.json에 표시된 해당 패키지를 업데이트해준다 보면된다.

package.json에 개발 당시의 version이 저장되어있기 때문에 node_module을 지우게 되더라도
문제없이 다시 실행할수있다.
