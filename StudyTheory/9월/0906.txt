MVC - 코드를 구조화하기 위한 특정 패턴
Model = 데이터작업을 처리 ex) 데이터 저장, 파일에서 데이터 가져오기등

Views = 사용자가 보는것을 처리. ex) html문서를 렌더링하고 다시 응답하는것.
        -> 생성을 위해서 템플릿 엔진에 주입하는 데이터, 컨트롤러
Controller = view 와 model의 중간 역할을 하는것. 작업할 모델과 렌더링할 뷰를 정의하는것.
             ex) Routes

오류) navigation.ejs랑 오류가 생긴다..
 해결 -> 404페이지를 렌더링할시에 path가 지정되있지않은 상태로 표현하게 된다.
        따라서 해당 페이지(404)에  path값을 넘겨줌으로써 해결했다.

클래스를 생성할시에 constructor를 통해서 만드는데 이후 만들어지는 함수들에 대해서 this를 사용하게되면
constructor를 통해서 만들어진 객체를 참조하게 된다.
때문에 class내부의 함수는 this를 사용하여 외부로 값을 빼낼수있다.

클래스를 다른곳에서 import할경우에 관례상 시작문자를 대문자로 표시하게 된다.
로컬 속성과 전역 속성을 구분해야된다.
